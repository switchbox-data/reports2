## Gaps

- **`zero_crossing` and `zero_crossing_ss`**: Both are present in `report_variables.RData` (they are assigned at chunk top level inside `fig-cumulative-saving` / `fig-cumulative-saving-ss`, so `save.image()` captures them). `zero_crossing` is `0.5517` (55.2%); however `zero_crossing_ss` is an empty `numeric(0)` because under the scattershot baseline the cumulative-saving curve never reaches zero, so the `filter(cumulative_saving <= 0) |> slice(1) |> pull(...)` returns nothing. This means `index.qmd` cannot reference `zero_crossing_ss` as a scalar without a guard.
- **Per-neighborhood summary statistics for Lincoln Square, Garfield Ridge, and Englewood**: Not present in the .RData. The neighborhoods chunk prints its summary inline without assigning the result to a named object, so nothing flows through `save.image()`. The only neighborhood-related artifacts saved are `target_areas` (the three names), `target_rows` (an unrelated numeric vector of plot positions), and `neighborhoods` (the full 98-row neighborhoods sf table) - none of which contain pre-computed cost/coverage stats for those three neighborhoods. To use neighborhood-specific numbers in `index.qmd`, the chunk would need explicit assignments (e.g. `englewood_stats <- ...`).
